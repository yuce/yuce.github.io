How To Install the NATS Server on Ubuntu 15.10

Introduction

NATS is a high performance messaging platform, often described as "nervous system of the internet".

Goals

  • Retrieve and install NATS 0.7.2.
  • Install a Let's Encrypt certificate to secure communication.
  • Cluster several NATS servers to increase reliability.

Prerequisites

  • 1 x small Ubuntu 15.10 droplet
  • (optionally) + 2x small Ubuntu 15.10 droplet (for clustering)
  • Complete steps 1-3 of Initial Server Setup with Ubuntu 14.04 to create the <^>demo<^> user

Obtaining and Installing the NATS Server

If you have correctly completed steps 1 to 3 of Initial Server Setup with Ubuntu 14.04 tutorial, you should have the <^>demo<^> user on your system.

Let's switch to the <^>demo<^> user, and change the directory to its home:

su - demo

We can now download gnats 0.7.2. You can also check nats.io/download for a later version:

wget https://github.com/nats-io/gnatsd/releases/download/v0.7.2/gnatsd-v0.7.2-linux-amd64.tar.gz

Extract the server executable:

tar xf gnatsd-v0.7.2-linux-amd64.tar.gz

On Linux, nonstandard software is usually kept at /opt directory; /opt/bin is a good place to keep the gnatsd executable. Let's first create the directory:

sudo mkdir -p /opt/bin

And then, move gnatsd to its place:

sudo mv gnatsd /opt/bin

Optionally, we can remove the files we don't need:

rm gnatsd-v0.7.2-linux-amd64.tar.gz LICENSE README.md

Let's test that we can run gnatsd:

/opt/bin/gnatsd

The output should be similar to this:

[1289] 2016/04/08 16:48:45.353086 [INF] Starting gnatsd version 0.7.2
[1289] 2016/04/08 16:48:45.353812 [INF] Listening for client connections on 0.0.0.0:4222
[1289] 2016/04/08 16:48:45.354497 [INF] gnatsd is ready

By default, gnatsd will listen on port 4222 and on all interfaces.

Press Ctrl+C to shutdown gnatsd to continue.

Adding a Systemd Entry

Systemd is a service manager for Linux systems. It is responsible for starting services on boot, restarting them as necessary and stopping in a controlled fashion on system shutdown. Systemd was created by Red Hat, and should be available on most RHEL and CentOS systems out there. It is available on Ubuntu 15.04+.

printf "[Unit]\nDescription=NATS server\n\n[Service]\nExecStart=/opt/bin/gnatsd\nRestart=on-abort\n\n[Install]\nWantedBy=multi-user.target\n" > gnatsd.service
sudo mv gnatsd.service /etc/systemd/system
